III. 화면 구현

1. UI 표준 및 지침
- 1. 정의
  = 1. UI 표준 -> 모든 UI 에 공통적으로 적용될 내용, 화면 구성, 화면 이동 포함
  = 2. UI 지침 -> UI 요구사항, 구현 시 제약사항, 공통 과정

- 2. UI 스타일 가이드
  = 1. 개발자 & 디자이너, UI 작성 시, 기준이 되는 규칙
  = 2. 구동 환경, 레이아웃, 네비게이션

- 3. UI 스타일 가이드 작성 순서
  = 1. 구동 환경 정의 -> 프레임 세트, OS, 웹 브라우저
  = 2. 레이아웃 정의 -> 화면 구조 정의 & 각 영역 메뉴 구성
  = 3. 네비게이션 정의 -> 네비게이션 메뉴 타입 선택, 적용
  = 4. 기능 정의 -> 데이터 관계를 논리적인 모델로 상세화
  = 5. 구성 요소 정의 -> 화면에 표시할 그리드 & 버튼 정의


2. UI 설계 도구
- 1. 와이어프레임 (Wireframe)
  = 1. 페이지에 대한 개략적인 레이아웃이나, 뼈대를 설계하는 도구
  = 2. 손그림, 파워포인트, 키노트, 스케치, 일러스트, 포토샵

- 2. 목업 (Mockup)
  = 1. 실제 화면과 유사하게 만든 정적인 형태의 모형
  = 2. 파워 목업, 발사믹 목업

- 3. 스토리보드 (Story Board)
  = 1. 와이어프레임에 콘텐츠에 대한 설명, 페이지 간 이동 흐름 등을 추가한 문서
  = 2. 서비스 구축을 위한 모든 정보가 들어 있음
  = 3. 파워포인트, 키노트, 스케치, Axure

- 4. 프로토타입 (Prototype)
  = 1. 실제 구현된 것처럼 테스트가 가능한 동적인 형태의 모형
  = 2. 종류
    -> 1. 페이퍼 프로토타입 => 손으로 직접 작성, 제작기간 짧고 적고, 업무 협의 빠름
    -> 2. 디지털 프로토타입 => 프로그램 사용, 재사용 필요할 때, 산출물과 비슷한 효과

- 5. 유스케이스
  = 1. 사용자의 요구사항을 기능 단위로 표현하는 것
  = 2. 다이어그램 형식, 프로젝트 초기에 시스템 기능적인 요구 결정, 결과 문서화 가능


3. UI 설계
- 1. UI 설계서
  = 1. 사용자의 요구사항을 바탕으로 UI 설계를 구체화하여 작성하는 문서
  = 2. 작성 순서
    -> 1. UI 설계서 표지 작성 => 다른 문서 혼동 X, 프로젝트명 & 시스템명 포함 작성
    -> 2. UI 설계서 개정 이력 작성 => 어떤 부분이 어떻게 수정되었는지 작성
    -> 3. UI 요구사항 정의서 작성 => 사용자 요구사항 확인 및 정리
    -> 4. 시스템 구조 작성 => UI 요구사항 & UI 프로토타입 기초, 전체 시스템 구조 설계
    -> 5. 사이트 맵 작성
       => 1. 사이트 맵 - 화면 정보, 시각적인 콘텐츠, 테이블 형태
       => 2. 사이트에 표시할 콘텐츠를 메뉴별로 구분하여 설계

    -> 6. 프로세스 정의서 작성 => 사용자 요구 프로세스, 작업 진행 순서에 맞춰 정리
    -> 7. 화면 설계 => 필요한 화면을 페이지별로 설계

- 2. UI 흐름 설계
  = 1. 업무의 진행과정, 수행 절차에 따른 흐름 파악, 화면과 폼을 설계하는 단계
  = 2. 흐름 설계 순서
    -> 1. 기능 작성 - 입력 요소 확인 - 유스케이스 설계 - 기능 및 양식 확인
    -> 2. 유스케이스 -> 화면에 표현할 입력 요소들 형태, 입력 방법, 배치 고려, 설계

- 3. UI 상세 설계
  = 1. UI 설계서, 실제 설계 및 구현, 모든 화면에 대해 자세하게 설계 진행
  = 2. 상세 설계 순서
    -> 1. 요구사항 확인 
    -> 2. UI 설계서 표지 & 개정 이력 작성
    -> 3. UI 구조 설계
    -> 4. 메뉴 구조 설계
    -> 5. 화면 설계

- 4. UI 시나리오 문서
  = 1. 사용자 인터페이스 기능 구조, 대표 화면, 화면 간 인터렉션의 흐름 정의한 문서
  = 2. UI 시나리오 문서 요건
    -> 완전성, 일관성, 이해성, 가독성, 수정 용이성, 추적 용이성


4. HCI / UX / 감성공학
- 1. HCI -> 사람이 시스템을 보다 편리, 안전하게 사용, 연구 및 개발하는 학문
- 2. UX 
  = 1. 사용자가 시스템이나 서비스 이용하면서 느끼고 생각하게 되는 총체적인 경험
  = 2. 특징
    -> 주관성 (개인적, 신체적, 인지적), 정황성 (경험), 총체성 (심리적, 감성적)
    
- 3. 감성공학 -> 제품이나 작업환경을 사용자의 감성에 알맞도록 설계 및 제작하는 기술
