II. UI 테스트

1. 사용자 인터페이스
- 1. 사용자와 시스템 간의 상호작용이 원활하게 이뤄지도록 도와주는 장치나 S/W
- 2. 사용자 인터페이스의 세 가지 분야
  = 1. 정보 제공 전달 - 물리적 제어
  = 2. 콘텐츠 상세적인 표현 & 전체적인 구성
  = 3. 모든 사용자가 편리하고 간편 사용하도록 하는 기능

- 3. 사용자 인터페이스의 구분
  = 1. CLI(Command Line Interface, 명령, 출력, 텍스트)
  = 2. GUI(Graphical User Interface, 아이콘 & 메뉴)
  = 3. NUI(Natural User Interface, 사용자 말 & 행동, 기기 조작)

- 4. 기본 원칙
  = 직관성(쉽게 이해), 유효성(정확), 학습성(배우고 익힘), 유연성(요구사항 수용, 실수 최소화)


2. 사용성 테스트 (Usability Test)
- 1. 일반 사용자가 사용하면서 문제점이나 개선점을 도출해 내는 방식의 테스트
- 2. 사용성
  = 1. 정의 -> 사용자가 정확히 이해하고 사용, 향후 다시 사용하고 싶은 정도
  = 2. Jakob Nielson -> 학습용이성, 사용효율성, 기억용이성, 최소한의 에러, 주관적 기쁨
  = 3. UPA (Usability Professionals Association) -> 회사 비용 절감, 사용자 요구 부합 제품. 시용자 피드백 받는 제품 개발 방식

- 3. 사용성 테스트 기법
  = 1. 파일럿 테스트 (Pilot Test) -> 구현 전, 소규모 시험 작동, 모의 시행
  = 2. NEM 기법 (Novice Expert radio Method)
    -> 제품 설계 및 개발 참여한 숙련자, 테스트 참여, 수행시간 비교 & 문제점 찾기
    
  = 3. 심층 인터뷰 (In Depth Interview)
    -> 사용자 일대일 면접 => 심리 파악, 조사원 숙련도따라 결과 신뢰성 & 타당성 변화

  = 4. 포커스 그룹 인터뷰 (FGI; Focus Group Interview)
    -> 사용자 6 ~ 12명, 면접자 진행, 조사 목적과 관련된 토론 진행

  = 5. 맥락적 인터뷰 (Contextual Inquiry / Interviews)
    -> 특정 상황, 맥락 정하기, 사용자 사용하는 동안 인터뷰 진행 & 의견 수집

- 4. 사용성 평가 분석서
  = 1. 정의 -> 결과 데이터 취합 & 분석 & 문서화화)
  = 2. 종류
    -> 1. 에러 메트릭스 => 사용자 범한 실수 수집 및 분석
    -> 2. 효율성 메트릭스 => 테스트 수행 시간, 수행한 행동 개수, 테스트 성공률 분석
    -> 3. 시간 기반 테스크 메트릭스 => 테스트 각 구간, 수행 시간 기록 & 분석


3. UI 테스트 기법의 종류
- 1. UI 테스트
  = 1. 구현된 UI 의 사용성을 검증하기 위해 테스트 수행
  = 2. 결과에 따라 개선 및 결과 보고서 작성하는 행위 또는 절차

- 2. 종류
  = 1. 휴리스틱 평가 (Heuristic Evaluation)
    -> 3명 이상 디자인 전문가, 전문가 능력 & Jakob Nielsen 10개 원칙
  
  = 2. 페이퍼 프로토타입 (Paper Prototype)
    -> 종이로 해당 서비스 간단 제작, 실제 구현처럼 표현, 테스트 기법 중 가장 빠름

  = 3. 선호도 평가 (Preference Evaluation)
    -> 선호도, 사용자 감성 분석 (과학적인 시점), 자료 특성 (점수, 순위)

  = 4. 성능 평가 (Performance Evaluation)
    -> 개발 마지막 단계, 제품의 학습성, 효율성, 기억용이성, 오류, 만족도
  
  = 5. 컨셉 모델 (Concept Model)
    -> 추상적인 컨셉, 관계 간편 시각화, UI 컨셉션, GUI 컨셉션

  = 6. 멘탈 모델 (Mental Model) -> 행동 동기 & 사고 과정, 다음 행동들의 패턴 분석
  = 7. 카드 소팅 (Card Sorting) -> 아이디어와 컨셉 카드에 작성, 분류 및 정렬 & 그룹화


4. UI 테스트 결과 보고
- 1. 사용성 & UI 테스트를 통해 도출된 결과에 대한 개선 방안 마련
- 2. 컨셉 모델, 멘탈 모델 기법 활용
- 3. UI 테스트 결과 보고 프로세스
  = 1. UI 개선 방안 및 수집 -> 도출된 결과 대상, 개선 필요한 항목 선정
  = 2. UI 개선 및 수정 수행 -> 개선 방안 및 수정 계획 바탕으로, UI 개선
  = 3. UI 개선 결과 보고서 작성 & 공유 -> ISO / IEC 9126 표준 활용


5. 품질 요구 사항
- 1. 사용자의 요구 얼마나 충족하는가를 나타내는 S/W 특성의 총체
- 2. S/W 품질 관련 표준
  = 1. ISO / IEC 9126 -> 품질 특성과 평가 (국제 표준)
  = 2. ISO / IEC 25010 -> ISO / IECT 9126 호환성 & 보안성강화, S/W 제품 (국제 표준)
  = 3. ISO / IEC 12119 -> 일반적인 제품 품질 요구사항 & 테스트 (국제 표준)
  = 4. ISO / IEC 14598 -> 품질 측정 & 평가에 필요 절차 규정 (표준)

- 3. ISO / IEC 9126 S/W 품질 특성
  = 1. 기능성 -> 적절/정합, 정밀/정확, 상호 운용, 보안, 준수 => 요구사항 정확 만족하는 기능 제공 여부
  = 2. 신뢰성 -> 성숙, 고장 허용, 회복 => 주어진 기능 오류 없이 수행
  = 3. 사용성 -> 이해, 학습, 운용, 친밀 => 사용자 정확하게 이해 사용, 재사용 희망 정도
  = 4. 효율성 -> 시간 효율, 자원 효율 => 요구하는 기능 빠르게 처리 가능 여부
  = 5. 유지 보수성 -> 분석, 변경, 아정, 시험 => S/W 개선 & 확장할 수 있는 정도 
  = 6. 이식성 -> 적용, 설치, 대체, 공존 => S/W 의 다른 환경에서의 쉽게 적용 여부
